1. Primary key: is a column (or set of columns) that uniquely identifies each row in a table.
-->properties:
1.Unique – No two rows can have the same value.
2.Not NULL – It cannot contain NULL values.
3.One primary key per table – But it can contain multiple columns (composite key).

2. Foreign Key: is a column that links one table to another table.(a colum that refers the primary key of other table).
-->It creates a relationship between tables and ensures data consistency.

SYNTAX:
FOREIGN KEY (column_name) REFERENCES parent_table(parent_column);
This means:

column_name → column in the current table
parent_table → table you are referencing
parent_column → primary key in the parent table

--------imp:Referential Integrity: It is a rule that ensures a foreign key value in one table must match an existing primary key value in another table.
Referential Integrity in a database means maintaining correct relationships between tables so that the data always stays valid and consistent.

In simple words:
If one table references another table, the referenced value must exist in the parent table.
(Think of it like a guest list at a party 
If a ticket refers to Guest ID 5, then Guest ID 5 must exist in the guest list. Otherwise the ticket is invalid.)

1.ON DELETE CASCADE :When a parent row is deleted, all related rows in the child table are automatically deleted.

USE SOCIAL_DB;
CREATE TABLE CUSTOMERSS (
    cust_id INT PRIMARY KEY,
    name VARCHAR(50)
);

CREATE TABLE ORDERSS (
    ord_id INT PRIMARY KEY,
    cust_id INT,
    FOREIGN KEY (cust_id)
    REFERENCES CUSTOMERSS(cust_id)
    ON DELETE CASCADE
);

Drop table orderss;
DROP TABLE CUSTOMER2;

(2,'Neha'),
(3,'Arjun');

-- INSERT IN ORDERSS TABLE
INSERT INTO ORDERSS VALUES
(101,1),
(102,1),
(103,2),
(104,3);

SELECT * FROM CUSTOMERSS;

SELECT * FROM ORDERSS;

-- dleting the cutsomer with id=1, will delete the order records related to cutomer having id=1(i.e, Ravi here)
DELETE FROM CUSTOMERSS
WHERE cust_id = 1;

SELECT * FROM CUSTOMERSS;
SELECT * FROM ORDERSS;
------------------>(rows with cust_id = 1 are automatically deleted), So orders of Ravi disappear automatically.

-------------------------------------------------2.ON DELETE SET NULL: When the parent row is deleted, the foreign key in the child table becomes NULL.
ON DELET NULL:If the parent row is deleted, the foreign key in the child table becomes NULL.

----------------------------------3. on DELETE RESTRICT:
ON DELETE RESTRICT means:
If a child record exists, you cannot delete the parent record.
The database restricts (blocks) the deletion.


ON DELET RESTRICT:
CREATE TABLE DEPARTMENTS (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50)
);
